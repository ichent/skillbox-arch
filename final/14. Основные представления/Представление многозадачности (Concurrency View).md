Описание:
Система использует гибридную модель обработки запросов для обеспечения высокой пропускной способности:
1. Синхронная обработка (Request/Response): Для блокирующих операций, требующих мгновенного ответа пользователю (авторизация, чтение профиля).
2. Асинхронная обработка (Event-Driven): Для тяжелых операций (аналитика тренировки, начисление наград). Используется брокер сообщений Apache Kafka. Это позволяет сервисам обрабатывать пиковые нагрузки параллельно, масштабируя количество потребителей (Consumers).
