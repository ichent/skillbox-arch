1. Архитектуцра представлена 2 частями: мобильное приложение и серверная часть
2. В мобильной части есть слой для работы с внешними датчиками, встроенными датчиками устройства и для работы с Phone Health API (если оно есть). Также там есть модули, которые отвечают за тренировки и профиль пользователя
3. Серверная часть представленая API Gateway, который принимает весь трафик. Архитектура построена в стиле SBA (service based architecture)
4. Есть модуль для аутентификации. Есть отдельный сервис со своей БД, который отвечает за тренировки. Также есть более крупный сервис со своей БД, который объединяет несколько модулей (Геймификация, Промо и реклама, Профили пользователей, Управление инвентарем)
5. Также есть отдельный сервис, который отвечает за уведомления. Например, отправляет PUSH-уведомления на устройство пользователя
