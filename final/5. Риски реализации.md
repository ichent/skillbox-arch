1. Бизнес-риски
    1. При росте популярности нужно будет увеличивать производство инвентаря
    2. Нужно будет держать качество инвентаря стабильно высоким
    3. Быть готовым нанимать больше специалистов (для рекламы, для создания новых видов инвентаря)
2. Технические риски
    1. Интеграция с устройствами может занять длительный процесс (их много, они могут отличаться в плане API)
    1.1 Для минимизации этих рисков на начальном этапе можно взять только наиболее популярные устройства (для них возможно уже будут написаны SDK для интеграции)
    2. Для проведения масштабных состязаний нужно быть готовым к нагрузкам
    2.1 К этому сервис будет готов, так как сервис, который отвечает за тренировки отдельный и его можно легко масштабировать.
    3. Модуль геймификации на текущий момент не описан и не ясна его функциональность
    3.1 Для улучшения этого нужно попробовать договориться с заказчиками на какой-то первоначальный вариант, который утсроит по времени и функционалу и реализовать его с учетом возможностей команды
    4. Нужно особенно относиться к безопасности приложения в плане работы с персональными данными
    4.1 Для поддержки этого нужно придерживаться подхода с зонами безопасности, чтобы доступ к данным был разграничен и данные были зашифрованы
