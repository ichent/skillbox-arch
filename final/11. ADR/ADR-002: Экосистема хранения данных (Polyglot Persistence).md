Статус: Принято.
Контекст: Единая реляционная СУБД (PostgreSQL) не может эффективно обрабатывать поток временных рядов (Time-Series) от IoT-устройств без деградации производительности транзакционных операций (заказы, инвентарь).
Решение: Применить гибридный подход к хранению данных:
    PostgreSQL: Для сервисов Inventory, Identity, Social (ACID-транзакции, реляционные связи).
    TimescaleDB: Для сервиса Training Core (хранение метрик с датчиков, оптимизация под time-series).
    Redis: Для кэширования сессий и Leaderboards (сервис Gamification).
Последствия:
    (+) Оптимальная производительность для каждого типа данных.
    (-) Необходимость поддержки нескольких типов СУБД администраторами.
