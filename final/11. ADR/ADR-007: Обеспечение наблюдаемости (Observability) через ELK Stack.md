Статус: Принято.

Контекст: В распределенной системе сложно отслеживать первопричину ошибок, так как логи разбросаны по разным контейнерам. Необходима централизованная агрегация логов.

Решение: Внедрить ELK Stack (Elasticsearch, Logstash, Kibana) для сбора и анализа логов, а также Prometheus + Grafana для сбора метрик производительности.

Последствия:
    (+) Единый интерфейс для поиска ошибок во всех сервисах.
    (+) Возможность настройки алертинга при аномалиях.
    (-) Требует значительных ресурсов хранения для индексов Elasticsearch.
