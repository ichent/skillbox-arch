Статус: Принято.
Контекст: Клиенты (мобильные приложения) не должны знать о внутренней топологии микросервисов. Необходима единая точка входа для контроля трафика.
Решение: Использовать паттерн API Gateway (реализация на базе Nginx или Kong).
Последствия:
    (+) Скрытие внутренней структуры системы.
    (+) Централизованная терминация SSL и Rate Limiting.
    (-) Появление единой точки отказа (требует развертывания в режиме High Availability).
