Статус: Принято.
Контекст: Ключевой сценарий использования приложения предполагает отсутствие стабильного интернет-соединения (тренировки в лесу/горах). Потеря данных недопустима (Reliability NFR).
Решение: Использовать локальную встраиваемую БД (SQLite/Realm) на мобильном устройстве как основной источник правды в момент тренировки. Реализовать механизм фоновой синхронизации (Background Sync) при восстановлении сети.
Последствия:
    (+) Гарантированная доступность функционала записи тренировки без сети.
    (-) Сложность реализации разрешения конфликтов синхронизации (Data Conflict Resolution).
